## 3. 表结构详细设计

### 3.1 user — 用户表

```sql
CREATE TABLE `user` (
    `id`           BIGINT        NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `username`     VARCHAR(50)   NOT NULL                COMMENT '用户名(登录)',
    `password`     VARCHAR(255)  NOT NULL                COMMENT '密码(BCrypt加密)',
    `nickname`     VARCHAR(100)  DEFAULT NULL            COMMENT '昵称',
    `email`        VARCHAR(100)  DEFAULT NULL            COMMENT '邮箱',
    `avatar`       VARCHAR(500)  DEFAULT NULL            COMMENT '头像URL',
    `role`         VARCHAR(20)   NOT NULL DEFAULT 'USER' COMMENT '角色: ADMIN / USER',
    `status`       TINYINT       NOT NULL DEFAULT 1      COMMENT '状态: 1启用 0禁用',
    `deleted`      TINYINT       NOT NULL DEFAULT 0      COMMENT '逻辑删除: 0未删 1已删',
    `create_time`  DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time`  DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_username` (`username`),
    UNIQUE KEY `uk_email` (`email`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| id | BIGINT | PK | 自增主键 |
| username | VARCHAR(50) | ✅ | 登录用户名，唯一 |
| password | VARCHAR(255) | ✅ | BCrypt 加密后的密码（60 字符，留 255 余量） |
| nickname | VARCHAR(100) | ❌ | 展示昵称 |
| email | VARCHAR(100) | ❌ | 邮箱，唯一 |
| avatar | VARCHAR(500) | ❌ | 头像文件路径或 URL |
| role | VARCHAR(20) | ✅ | 角色标识，ADMIN 或 USER |
| status | TINYINT | ✅ | 账号状态，1=启用，0=禁用 |
| deleted | TINYINT | ✅ | 逻辑删除标志 |

### 3.2 knowledge — 知识库表

```sql
CREATE TABLE `knowledge` (
    `id`              BIGINT        NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `user_id`         BIGINT        NOT NULL                COMMENT '创建者ID',
    `name`            VARCHAR(100)  NOT NULL                COMMENT '知识库名称',
    `description`     VARCHAR(500)  DEFAULT NULL            COMMENT '知识库描述',
    `icon`            VARCHAR(200)  DEFAULT NULL            COMMENT '图标URL',
    `is_public`       TINYINT       NOT NULL DEFAULT 0      COMMENT '是否公开: 0私有 1公开',
    `doc_count`       INT           NOT NULL DEFAULT 0      COMMENT '文档数量(冗余)',
    `chunk_count`     INT           NOT NULL DEFAULT 0      COMMENT '分块总数(冗余)',
    `embedding_model` VARCHAR(50)   DEFAULT 'text-embedding-3-small' COMMENT 'Embedding模型标识',
    `status`          TINYINT       NOT NULL DEFAULT 1      COMMENT '状态: 1正常 0禁用',
    `deleted`         TINYINT       NOT NULL DEFAULT 0      COMMENT '逻辑删除: 0未删 1已删',
    `create_time`     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time`     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='知识库表';
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| user_id | BIGINT | ✅ | 外键关联 user.id |
| name | VARCHAR(100) | ✅ | 知识库名称 |
| description | VARCHAR(500) | ❌ | 知识库描述，展示用途 |
| is_public | TINYINT | ✅ | 0=私有(仅创建者可访问)，1=公开(所有人可读) |
| doc_count | INT | ✅ | 文档数量，冗余字段，异步更新 |
| chunk_count | INT | ✅ | 分块总数，冗余字段 |
| embedding_model | VARCHAR(50) | ✅ | 创建时选定的 Embedding 模型，后续不可更改 |

### 3.3 document — 文档表

```sql
CREATE TABLE `document` (
    `id`            BIGINT        NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `knowledge_id`  BIGINT        NOT NULL                COMMENT '所属知识库ID',
    `file_name`     VARCHAR(255)  NOT NULL                COMMENT '原始文件名',
    `file_type`     VARCHAR(20)   NOT NULL                COMMENT '文件类型: pdf/docx/txt/md/csv',
    `file_size`     BIGINT        NOT NULL                COMMENT '文件大小(字节)',
    `file_path`     VARCHAR(500)  NOT NULL                COMMENT '存储路径(相对路径)',
    `chunk_count`   INT           NOT NULL DEFAULT 0      COMMENT '分块数量',
    `char_count`    INT           NOT NULL DEFAULT 0      COMMENT '字符总数',
    `status`        TINYINT       NOT NULL DEFAULT 0      COMMENT '状态: 0待处理 1解析中 2向量化中 3已完成 4失败',
    `error_msg`     VARCHAR(500)  DEFAULT NULL            COMMENT '失败原因',
    `deleted`       TINYINT       NOT NULL DEFAULT 0      COMMENT '逻辑删除: 0未删 1已删',
    `create_time`   DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time`   DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    KEY `idx_knowledge_id` (`knowledge_id`),
    KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文档表';
```

**状态机：**

```
    ┌─────────┐
    │ 0 待处理 │ ◀─── 上传完成
    └────┬────┘
         │ 开始解析
         ▼
    ┌─────────┐
    │ 1 解析中 │
    └────┬────┘
         │ 解析完成
         ▼
    ┌──────────┐
    │ 2 向量化中│
    └────┬─────┘
         │ 向量化完成
         ▼
    ┌──────────┐
    │ 3 已完成  │ ◀─── 正常终态
    └──────────┘

    任何阶段失败 ──▶ 4 失败（记录 error_msg）
```

### 3.4 document_chunk — 文档分块表

```sql
CREATE TABLE `document_chunk` (
    `id`            BIGINT        NOT NULL AUTO_INCREMENT  COMMENT '主键ID',
    `document_id`   BIGINT        NOT NULL                 COMMENT '所属文档ID',
    `knowledge_id`  BIGINT        NOT NULL                 COMMENT '所属知识库ID(冗余，方便检索)',
    `chunk_index`   INT           NOT NULL                 COMMENT '分块在文档中的序号(从0开始)',
    `content`       TEXT          NOT NULL                 COMMENT '分块文本内容',
    `token_count`   INT           DEFAULT NULL             COMMENT 'Token 数量',
    `char_count`    INT           NOT NULL DEFAULT 0       COMMENT '字符数量',
    `vector_id`     VARCHAR(100)  DEFAULT NULL             COMMENT '向量存储中的ID',
    `metadata`      JSON          DEFAULT NULL             COMMENT '元数据: {"page":1,"title":"第一章"}',
    `deleted`       TINYINT       NOT NULL DEFAULT 0       COMMENT '逻辑删除: 0未删 1已删',
    `create_time`   DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP  COMMENT '创建时间',
    PRIMARY KEY (`id`),
    KEY `idx_document_id` (`document_id`),
    KEY `idx_knowledge_id` (`knowledge_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文档分块表';
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| chunk_index | INT | 分块序号，用于排序还原 |
| content | TEXT | 分块的纯文本，用于关键词检索和引用展示 |
| token_count | INT | Embedding 模型的 token 消耗 |
| vector_id | VARCHAR(100) | 向量存储中的唯一标识，用于关联向量记录 |
| metadata | JSON | 结构化元数据，不同文档类型存放不同信息 |

**metadata 示例：**

```json
// PDF 文档
{"page": 5, "title": "第三章 Spring AI 配置"}

// Markdown 文档
{"heading": "## 安装指南", "level": 2}

// CSV 文档
{"row_range": "100-150", "columns": ["name","age","city"]}
```

### 3.5 chat_session — 对话会话表

```sql
CREATE TABLE `chat_session` (
    `id`            BIGINT        NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `user_id`       BIGINT        NOT NULL                COMMENT '用户ID',
    `knowledge_id`  BIGINT        DEFAULT NULL            COMMENT '关联知识库ID(NULL表示通用对话)',
    `title`         VARCHAR(200)  NOT NULL DEFAULT '新对话' COMMENT '会话标题',
    `model_name`    VARCHAR(50)   NOT NULL DEFAULT 'gpt-4o-mini' COMMENT '使用的模型',
    `temperature`   DECIMAL(2,1)  NOT NULL DEFAULT 0.7    COMMENT '温度参数 0.0-2.0',
    `top_k`         INT           NOT NULL DEFAULT 5      COMMENT 'RAG检索Top-K',
    `max_history`   INT           NOT NULL DEFAULT 10     COMMENT '携带历史轮数',
    `deleted`       TINYINT       NOT NULL DEFAULT 0      COMMENT '逻辑删除: 0未删 1已删',
    `create_time`   DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time`   DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    KEY `idx_user_id` (`user_id`),
    KEY `idx_knowledge_id` (`knowledge_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='对话会话表';
```

**字段说明：**

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| knowledge_id | BIGINT | NULL | NULL=通用对话，非NULL=RAG对话 |
| model_name | VARCHAR(50) | gpt-4o-mini | 每个会话可选不同模型 |
| temperature | DECIMAL(2,1) | 0.7 | 控制回答随机性 |
| top_k | INT | 5 | RAG 检索返回的文档块数量 |
| max_history | INT | 10 | 上下文窗口携带的历史消息轮数 |

### 3.6 chat_message — 对话消息表

```sql
CREATE TABLE `chat_message` (
    `id`            BIGINT        NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `session_id`    BIGINT        NOT NULL                COMMENT '会话ID',
    `role`          VARCHAR(20)   NOT NULL                COMMENT '角色: user / assistant / system',
    `content`       TEXT          NOT NULL                COMMENT '消息内容',
    `token_count`   INT           DEFAULT NULL            COMMENT 'Token 消耗量',
    `references`    JSON          DEFAULT NULL            COMMENT '引用来源(仅assistant): [{"chunkId":1,"content":"...","score":0.92}]',
    `create_time`   DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (`id`),
    KEY `idx_session_id` (`session_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='对话消息表';
```

**references 字段示例（仅 assistant 角色的消息有此字段）：**

```json
[
    {
        "chunkId": 42,
        "documentId": 7,
        "fileName": "SpringAI-Guide.pdf",
        "content": "Spring AI 提供了统一的 ChatClient 接口...",
        "score": 0.9231
    },
    {
        "chunkId": 108,
        "documentId": 12,
        "fileName": "RAG原理.md",
        "content": "RAG 的核心流程包括检索和生成两个阶段...",
        "score": 0.8876
    }
]
```

## 4. 索引策略

## 5. 设计规范

### 5.1 通用字段约定

所有业务表包含以下公共字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT AUTO_INCREMENT | 主键 |
| deleted | TINYINT DEFAULT 0 | 逻辑删除（MyBatis Plus @TableLogic） |
| create_time | DATETIME | 创建时间（MyBatis Plus @TableField(fill=INSERT)） |
| update_time | DATETIME | 更新时间（MyBatis Plus @TableField(fill=INSERT_UPDATE)） |

### 5.2 命名规范

| 项目 | 规范 | 示例 |
|------|------|------|
| 表名 | 小写下划线 | `document_chunk` |
| 字段名 | 小写下划线 | `create_time` |
| 主键 | `id` | BIGINT AUTO_INCREMENT |
| 外键 | `{关联表}_id` | `user_id`, `knowledge_id` |
| 逻辑删除 | `deleted` | TINYINT, 0=未删, 1=已删 |
| 索引 | `idx_{字段名}` | `idx_user_id` |
| 唯一索引 | `uk_{字段名}` | `uk_username` |

### 5.3 JSON 字段使用规范

| 表 | 字段 | 用途 | 频繁查询 |
|---|------|------|:---:|
| document_chunk | metadata | 存储结构化元数据（页码、标题） | ❌ |
| chat_message | references | 存储 RAG 引用来源列表 | ❌ |

原则：JSON 字段只存储**不参与 WHERE 条件**的数据，避免影响索引效率。
