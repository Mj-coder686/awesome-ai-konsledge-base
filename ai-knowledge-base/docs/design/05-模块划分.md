## 4. module/auth — 认证模块

### 职责边界

处理用户认证相关的所有操作：注册、登录、Token 刷新、退出。认证逻辑从 user 模块独立出来，user 模块只负责用户信息的 CRUD。

### 包结构

```
module/auth/
├── controller/AuthController.java       // 注册/登录/刷新/退出 API
├── service/AuthService.java             // 认证业务逻辑接口
├── service/impl/AuthServiceImpl.java    // 认证业务逻辑实现
└── domain/
    ├── dto/RegisterDTO.java             // 注册请求参数
    ├── dto/LoginDTO.java                // 登录请求参数
    └── vo/LoginVO.java                  // 登录响应(token+用户信息)
```

### 核心逻辑

```
注册: DTO校验 → 检查用户名唯一 → BCrypt加密 → 插入数据库 → 生成Token → 返回
登录: DTO校验 → 查询用户 → BCrypt验证 → 检查状态 → 生成Token → 缓存Redis → 返回
刷新: 验证旧Token → 生成新Token → 更新Redis → 返回
退出: 删除Redis中的Token → 返回
```

### 文档处理流程（异步）

```
Controller 接收文件
    │
    ├── 同步: 保存文件到磁盘 → 创建 Document 记录(status=0) → 返回响应
    │
    └── 异步(@Async): 
        ├── 更新 status=1 (解析中)
        ├── DocParserService 解析 → 纯文本
        ├── DocChunkService 分块 → List<Chunk>
        ├── 更新 status=2 (向量化中)
        ├── 调用 ai/EmbeddingService 向量化 → 写入 VectorStore
        ├── 批量写入 document_chunk 表
        ├── 更新 doc_count / chunk_count
        └── 更新 status=3 (完成) 或 status=4 (失败)
```

### 核心类职责

#### ChatService — 对话编排器

```
职责: 接收用户消息，判断对话类型，编排 RAG 或普通对话流程

普通对话:
  1. 从 Redis 加载最近 N 轮历史
  2. 组装 Prompt (system + history + user)
  3. 调用 ChatClient.stream() 获取 Flux
  4. 返回 SSE 流

RAG 对话:
  1. 从 Redis 加载最近 N 轮历史
  2. 调用 RagService.retrieve() 检索相关文档块
  3. 组装 Prompt (system + context + history + user)
  4. 调用 ChatClient.stream() 获取 Flux
  5. 持久化消息 + 引用来源
  6. 返回 SSE 流（附带引用信息）
```

#### RagService — RAG 检索器

```
职责: 执行 RAG 检索，返回最相关的文档分块

方法:
  retrieve(query, knowledgeId, topK) → List<ChunkResult>

流程:
  1. 调用 EmbeddingService 将 query 向量化
  2. 调用 VectorStoreService.similaritySearch()
     - filter: knowledge_id = {knowledgeId}
     - topK: 可配置
  3. 返回结果（含相似度分数）
```

#### EmbeddingService — 向量化服务

```
职责: 文本到向量的转换

方法:
  embed(text) → float[]
  batchEmbed(texts) → List<float[]>

实现: 封装 Spring AI EmbeddingModel，处理批量和重试
```

#### VectorStoreService — 向量存储抽象

```
职责: 向量的增删查

方法:
  add(chunks)          // 文档入库
  search(query, filter, topK) → List<ChunkResult>  // 相似性检索
  delete(filter)       // 按条件删除

设计: 封装 Spring AI VectorStore 接口
  Phase 1: SimpleVectorStore (文件存储)
  Phase 2: 切换为 PGVector / Milvus（只需换配置）
```

## 9. 模块间调用关系

```
                    ┌───────────┐
                    │   admin   │
                    └─────┬─────┘
                          │ 调用 user 查询
                          ▼
┌────────┐  注册查询  ┌───────────┐
│  auth  │ ─────────▶│   user    │
└────────┘           └───────────┘

┌──────────────┐  文档入库时调用Embedding  ┌───────────┐
│  knowledge   │ ───────────────────────▶│    ai     │
└──────────────┘                         └───────────┘
                                              │
┌──────────────┐  RAG对话时查询知识库         │
│  knowledge   │ ◀───────────────────────────┘
└──────────────┘

所有模块 ──▶ common (R, SecurityUtil, BusinessException)
所有模块 ──▶ config (拦截器、过滤器生效)
```

**设计原则：**
- 模块间通过 Service 接口调用，不直接调用 Mapper
- knowledge → ai 的依赖通过接口解耦（EmbeddingService 接口在 ai 模块定义）
- 禁止循环依赖，如需要则提取到 common 层
