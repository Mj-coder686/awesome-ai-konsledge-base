## 二、数据库（3 张表，练手够用）

```sql
CREATE DATABASE IF NOT EXISTS ai_knowledge_base DEFAULT CHARSET utf8mb4;
USE ai_knowledge_base;

-- 1. 用户表
CREATE TABLE `user` (
    `id`          BIGINT       PRIMARY KEY AUTO_INCREMENT,
    `username`    VARCHAR(50)  NOT NULL UNIQUE,
    `password`    VARCHAR(255) NOT NULL,
    `nickname`    VARCHAR(100) DEFAULT NULL,
    `create_time` DATETIME     DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 2. 知识库表
CREATE TABLE `knowledge` (
    `id`          BIGINT       PRIMARY KEY AUTO_INCREMENT,
    `user_id`     BIGINT       NOT NULL,
    `name`        VARCHAR(100) NOT NULL,
    `description` VARCHAR(500) DEFAULT NULL,
    `create_time` DATETIME     DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX `idx_user_id` (`user_id`)
);

-- 3. 文档表
CREATE TABLE `document` (
    `id`            BIGINT       PRIMARY KEY AUTO_INCREMENT,
    `knowledge_id`  BIGINT       NOT NULL,
    `file_name`     VARCHAR(255) NOT NULL,
    `file_type`     VARCHAR(20)  NOT NULL,
    `file_size`     BIGINT       NOT NULL,
    `file_path`     VARCHAR(500) NOT NULL,
    `status`        TINYINT      DEFAULT 0 COMMENT '0待处理 1已完成',
    `create_time`   DATETIME     DEFAULT CURRENT_TIMESTAMP,
    `update_time`   DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX `idx_knowledge_id` (`knowledge_id`)
);
```

**为什么是 3 张：**
- `user` — 练登录注册 + JWT
- `knowledge` — 练标准 CRUD + 分页查询
- `document` — 练文件上传 + 异步处理

后期加 RAG 时再加 `document_chunk` 表，当前不需要。

## 三、API 接口（8 个，全部是练手用）

| # | 方法 | 路径 | 练什么 |
|:-:|------|------|--------|
| 1 | POST | `/api/register` | 参数校验、密码加密、插入数据库 |
| 2 | POST | `/api/login` | 查询数据库、密码比对、生成 JWT |
| 3 | POST | `/api/knowledge` | 标准 POST 创建、JWT 取当前用户 |
| 4 | GET | `/api/knowledge` | 分页查询、MyBatis Plus Page |
| 5 | DELETE | `/api/knowledge/{id}` | 路径参数、级联删除 |
| 6 | POST | `/api/knowledge/{id}/documents` | 文件上传 multipart |
| 7 | GET | `/api/knowledge/{id}/documents` | 关联查询 |
| 8 | GET | `/api/documents/{id}` | 单条查询详情 |

**后期加 AI 对话时再加 `/api/chat`，现在不加。**

## 五、每天学什么、写什么

### Day 1 — 项目启动 + 数据库连接 + Entity

**今天练：** pom.xml、application.yml、MyBatis Plus 注解、Lombok

```
写这些文件：
  pom.xml                    — 调整依赖（去掉用不上的，加上 MyBatis Plus）
  application.yml            — 配 MySQL 连接
  schema.sql                 — 执行建表
  User.java                  — @Data @TableName @TableId
  Knowledge.java             — 同上
  Document.java              — 同上
  UserMapper.java            — extends BaseMapper<User>
  KnowledgeMapper.java
  DocumentMapper.java
  MyBatisPlusConfig.java     — 分页插件
```

**Day 1 结束后你应该会：**
- 独立写一个 Entity 类（知道 @Data、@TableName、@TableId 的作用）
- 配置 MySQL 数据源
- 用 BaseMapper 的内置方法做简单查询

---

### Day 2 — 注册登录 + JWT + 拦截器

**今天练：** Service 层、JWT 生成/解析、Filter 拦截器、Redis 存 Token

```
写这些文件：
  R.java                     — 统一返回体
  GlobalExceptionHandler.java — @RestControllerAdvice
  JwtUtil.java               — 生成Token / 解析Token / 验证Token
  JwtFilter.java             — 从Header取Token → 解析 → 放入request
  CorsConfig.java            — 允许跨域
  UserService.java           — 注册(密码加密) + 登录(验证密码)
  AuthController.java        — POST /register + POST /login
```

**Day 2 结束后你应该会：**
- 写 Service 接口和实现类
- 用 BCrypt 加密密码
- 用 jjwt 生成和解析 JWT
- 手写一个 Filter 拦截请求
- 理解请求从 Filter → Controller → Service → Mapper 的完整链路

---

### Day 3 — 知识库 CRUD + 分页

**今天练：** CRUD 完整链路、分页查询、LambdaQueryWrapper

```
写这些文件：
  KnowledgeService.java      — 创建 / 列表(分页) / 详情 / 删除
  KnowledgeController.java   — 对应 4 个接口（实际 MVP 只用创建+列表+删除）
```

**Day 3 结束后你应该会：**
- 用 MyBatis Plus 的 Page<T> 做分页
- 用 LambdaQueryWrapper 构建查询条件
- 从 JWT 中取当前用户 ID 做数据隔离
- 理解 Controller → Service → Mapper 的完整 CRUD 流程

---

### Day 4 — 文件上传 + 项目收尾

**今天练：** 文件上传、关联查询、Redis 基本操作

```
写这些文件：
  DocumentService.java       — 上传文件 / 按知识库查询 / 详情
  DocumentController.java    — 对应 3 个接口
  Redis 的简单使用            — 登录成功后把 Token 存 Redis，拦截器里从 Redis 校验
```

**Day 4 结束后你应该会：**
- 用 MultipartFile 接收上传文件
- 把文件存到本地目录
- 理解 Redis 的 set / get / delete 基本操作
- 用 Redis 做 Token 白名单（可选，练手用）
