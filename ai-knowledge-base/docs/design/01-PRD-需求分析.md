## 1. 产品概述

### 1.1 产品定位

AI Knowledge Base 是一个基于 **Spring Boot + Spring AI + RAG** 的智能知识库系统。用户可以上传文档建立私有知识库，通过 AI 对话的方式从知识库中精准获取信息，实现"让文档会说话"。

### 1.2 产品愿景

构建一个生产级的 AI 知识库系统，作为个人学习项目和求职作品集，展示：
- 全栈 Java 后端开发能力
- Spring AI / RAG 工程化落地能力
- 系统架构设计能力

### 1.3 目标用户

| 用户类型 | 场景 | 核心诉求 |
|---------|------|---------|
| **个人用户** | 学习笔记、技术文档管理 | 快速检索、AI 问答 |
| **企业场景** | 内部知识库、产品文档 | 多人协作、权限管控 |

## 2. 功能需求

### 2.1 功能全景图

```
┌─────────────────────────────────────────────────────────┐
│                    AI Knowledge Base                      │
├──────────┬──────────┬──────────────┬──────────┬──────────┤
│  用户中心  │ 知识库管理 │  文档管理     │  AI 对话  │  系统管理  │
│          │          │              │          │          │
│ · 注册登录 │ · 创建库  │ · 上传文档    │ · 普通对话 │ · 仪表盘  │
│ · 个人信息 │ · 编辑库  │ · 文档解析    │ · RAG对话  │ · 用户管理 │
│ · JWT认证 │ · 删除库  │ · 分块向量化  │ · 历史记录 │ · 系统配置 │
│          │ · 列表    │ · 状态监控    │ · 引用溯源 │ · 日志    │
└──────────┴──────────┴──────────────┴──────────┴──────────┘
```

### 2.2 功能详细说明

#### 模块一：用户中心

| 功能 | 优先级 | 说明 |
|------|:------:|------|
| 用户注册 | P0 | 用户名 + 密码注册，密码 BCrypt 加密 |
| 用户登录 | P0 | 登录获取 JWT Token，Token 有效期可配置 |
| 个人信息 | P1 | 查看/修改昵称、头像、邮箱 |
| 修改密码 | P1 | 旧密码验证 + 新密码设置 |
| Token 刷新 | P0 | 无感刷新，过期前自动续期 |

#### 模块二：知识库管理

| 功能 | 优先级 | 说明 |
|------|:------:|------|
| 创建知识库 | P0 | 名称 + 描述 + 选择 Embedding 模型 |
| 知识库列表 | P0 | 分页查询，支持名称搜索 |
| 编辑知识库 | P1 | 修改名称、描述 |
| 删除知识库 | P0 | 级联删除所有文档和向量（需二次确认） |
| 知识库统计 | P1 | 文档数、分块数、存储占用 |

#### 模块三：文档管理

| 功能 | 优先级 | 说明 |
|------|:------:|------|
| 文档上传 | P0 | 支持 PDF / DOCX / TXT / MD / CSV |
| 文档解析 | P0 | 自动提取文本内容（Apache Tika） |
| 文本分块 | P0 | 按语义分块（可配置 chunk size / overlap） |
| 向量化入库 | P0 | Embedding → 存入向量存储 |
| 文档列表 | P0 | 按知识库查看，支持状态筛选 |
| 文档删除 | P0 | 同步清理向量 |
| 处理状态 | P0 | 待处理 / 处理中 / 已完成 / 失败（状态机） |
| 文档预览 | P2 | 在线预览文档原文内容 |

#### 模块四：AI 对话（核心）

| 功能 | 优先级 | 说明 |
|------|:------:|------|
| 普通对话 | P0 | 不关联知识库，纯 LLM 对话 |
| RAG 对话 | P0 | 关联知识库，检索增强生成 |
| 流式输出 | P0 | SSE 实时返回，打字机效果 |
| 对话会话 | P0 | 多轮对话，会话管理 |
| 对话历史 | P1 | 持久化存储，可回溯 |
| 引用溯源 | P1 | 答案标注来源文档及段落 |
| 会话重命名 | P2 | 自定义会话标题 |
| 会话删除 | P2 | 删除会话及消息 |

#### 模块五：系统管理（后台）

| 功能 | 优先级 | 说明 |
|------|:------:|------|
| 仪表盘 | P1 | 用户数、知识库数、文档数、对话数统计 |
| 用户管理 | P1 | 用户列表、启用/禁用、角色分配 |
| 模型配置 | P1 | 配置可用的 AI 模型及参数 |
| 操作日志 | P2 | 记录关键操作 |

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 对话首字响应 | < 2s（流式首 token） |
| RAG 检索延迟 | < 500ms |
| 文档上传 | 支持单文件最大 50MB |
| 并发用户 | 初期支持 50 并发 |
| 向量检索 | Top-5 < 200ms |

### 3.2 安全需求

| 项目 | 要求 |
|------|------|
| 认证 | JWT Token 认证 |
| 密码 | BCrypt 加密存储 |
| 接口鉴权 | 除登录注册外所有接口需认证 |
| 数据隔离 | 用户只能访问自己的知识库 |
| SQL 注入 | MyBatis Plus 参数化查询 |
| CORS | 可配置的跨域策略 |

### 3.3 可扩展性

- AI 模型可切换（OpenAI / Ollama / 国产大模型）
- 向量存储可升级（Simple → Milvus / PGVector）
- 文档解析器可扩展（支持更多文件类型）
- 后端可水平扩展（无状态设计 + Redis 共享会话）

## 4. 用户故事 (User Stories)

### 4.1 P0 核心故事

```
US-01  作为一个新用户，我想注册账号并登录，以便开始使用系统。
       验收标准：注册后自动登录，返回 JWT Token。

US-02  作为一个用户，我想创建一个知识库并上传文档，以便建立我的私有知识。
       验收标准：上传后文档自动解析、分块、向量化，状态变为"已完成"。

US-03  作为一个用户，我想选择一个知识库进行 AI 对话，以便精准获取文档中的信息。
       验收标准：AI 回答基于检索到的文档内容，并标注来源。

US-04  作为一个用户，我想进行普通 AI 对话（不关联知识库），以便获得通用 AI 助手能力。
       验收标准：多轮对话，上下文连贯，流式输出。

US-05  作为一个用户，我想查看历史对话记录，以便回顾之前的问题和答案。
       验收标准：按会话分组，支持翻页。
```

### 4.2 P1 增强故事

```
US-06  作为一个用户，我想看到答案引用了哪些文档片段，以便验证 AI 回答的可靠性。
       验收标准：答案中显示 [1][2] 标注，hover 可查看原文。

US-07  作为一个管理员，我想查看系统使用统计，以便了解平台运营状况。
       验收标准：仪表盘显示用户数、知识库数、文档数、对话数趋势。

US-08  作为一个用户，我想配置对话参数（温度、模型），以便控制 AI 的回答风格。
       验收标准：每个会话可独立配置。
```

## 6. 术语表

| 术语 | 说明 |
|------|------|
| **RAG** | Retrieval-Augmented Generation，检索增强生成 |
| **Embedding** | 将文本转换为高维向量的过程 |
| **Chunk** | 文档分块，向量化的最小单元 |
| **Vector Store** | 向量数据库，存储和检索向量 |
| **SSE** | Server-Sent Events，服务端推送事件，用于流式输出 |
| **Token** | LLM 处理文本的基本单位 |
| **LLM** | Large Language Model，大语言模型 |
| **Agent** | 具备工具调用能力的 AI 智能体 |
