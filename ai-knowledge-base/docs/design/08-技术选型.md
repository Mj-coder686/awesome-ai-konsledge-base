## 1. 核心框架选型

### 1.1 Java 版本：21

| 选项 | 评估 |
|------|------|
| Java 17 (当前 pom.xml) | LTS，生态成熟，但 Spring Boot 3.5 已原生支持 21 |
| **Java 21** ✅ | 最新 LTS，虚拟线程 (Virtual Threads)、Pattern Matching、Record 等特性；Spring Boot 3.2+ 完全支持；简历加分项 |

**决策：** 使用 Java 21。当前 pom.xml 改为 `<java.version>21</java.version>`。

**风险：** 低。Spring Boot 3.5.x 已完全兼容 Java 21。

---

### 1.2 Spring Boot 3.5.x

| 选项 | 评估 |
|------|------|
| Spring Boot 3.4.x | 稳定但不是最新 |
| **Spring Boot 3.5.x** ✅ | 当前项目版本，最新稳定版，Spring AI 1.0.x 兼容 |

**决策：** 保持当前 Spring Boot 3.5.16 不变。

---

### 1.3 ORM：MyBatis Plus 3.5.x

| 选项 | 评估 |
|------|------|
| MyBatis (当前 pom.xml) | 基础版，需手写 CRUD SQL |
| **MyBatis Plus 3.5.x** ✅ | 在 MyBatis 基础上增强，内置 CRUD、分页、逻辑删除、自动填充；减少样板代码 |

**决策：** 从 `mybatis-spring-boot-starter` 替换为 `mybatis-plus-spring-boot-starter`。

**替换方案：**
```xml
<!-- 移除 -->
<dependency>
    <groupId>org.mybatis.spring.boot</groupId>
    <artifactId>mybatis-spring-boot-starter</artifactId>
</dependency>

<!-- 替换为 -->
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-spring-boot3-starter</artifactId>
    <version>3.5.7</version>
</dependency>
```

**MyBatis Plus 带来的收益：**

| 功能 | 说明 |
|------|------|
| BaseMapper | 内置 CRUD，无需为每个表写基础 SQL |
| @TableLogic | 逻辑删除自动处理 |
| @TableField(fill) | create_time / update_time 自动填充 |
| Page<T> | 分页查询内置支持 |
| LambdaQueryWrapper | 类型安全的条件构造器 |
| 代码生成器 | 按需使用，加速开发 |

---

### 1.4 Spring AI 1.0.x

| 选项 | 评估 |
|------|------|
| LangChain4j | Java 版 LangChain，社区活跃，但非 Spring 原生 |
| **Spring AI** ✅ | Spring 官方 AI 框架，与 Spring Boot 无缝集成，抽象层统一（Chat/Embedding/VectorStore） |

**决策：** 使用 Spring AI。

**关键依赖：**
```xml
<!-- Spring AI BOM (统一版本管理) -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.ai</groupId>
            <artifactId>spring-ai-bom</artifactId>
            <version>1.0.0</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<!-- OpenAI 兼容接口 (支持 OpenAI / Ollama / 通义千问等) -->
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-openai-spring-boot-starter</artifactId>
</dependency>
```

**为什么选 Spring AI 而不是 LangChain4j：**

| 维度 | Spring AI | LangChain4j |
|------|-----------|-------------|
| Spring 生态 | 原生集成 | 需要适配 |
| 配置方式 | application.yml | Java Config |
| 抽象层 | ChatClient / EmbeddingModel / VectorStore | ChatLanguageModel / EmbeddingStore |
| 流式支持 | Flux<String> 原生 | 需要额外处理 |
| Advisor 机制 | 内置拦截器链 | 需要手动编排 |
| 简历价值 | 更贴合 Spring 技术栈 | 偏 Python 生态思维 |

### 1.5 向量存储

| 阶段 | 选型 | 说明 |
|------|------|------|
| **Phase 1: SimpleVectorStore** ✅ | Spring AI 内置 | 零外部依赖，基于内存/JSON 文件，开发验证用 |
| Phase 2: PGVector | PostgreSQL 插件 | 生产可用，只需一个 PostgreSQL 实例 |
| Phase 3: Milvus | 独立向量数据库 | 大规模生产，需要额外部署 Milvus |

**决策：** 第一阶段使用 SimpleVectorStore，接口抽象已由 Spring AI VectorStore 统一，后续切换仅需更换依赖和配置。

**SimpleVectorStore 配置：**
```java
@Bean
public VectorStore vectorStore(EmbeddingModel embeddingModel) {
    return new SimpleVectorStore(embeddingModel);
    // 数据持久化到 JSON 文件
    // 后续切换: return new MilvusVectorStore(...)
}
```

## 2. 基础设施选型

### 2.1 数据库：MySQL 8.x

| 选项 | 评估 |
|------|------|
| **MySQL 8.x** ✅ | 最广泛使用的关系数据库，生态成熟，免费 |
| PostgreSQL 16 | 功能更强大（JSON、全文检索、PGVector），但学习成本略高 |
| MariaDB | MySQL 兼容分支，但生态不如 MySQL 本体 |

**决策：** 保持 MySQL 8.x。如果后续需要 PGVector，可额外部署 PostgreSQL 专门用于向量存储。

---

### 2.2 缓存：Redis 7.x

| 用途 | 说明 |
|------|------|
| JWT Token 白名单 | 存储有效 Token，支持主动注销 |
| 对话上下文缓存 | 最近 N 轮对话消息，减少数据库查询 |
| 频率限制 | 滑动窗口计数器 |
| 统计数据缓存 | Dashboard 数据，定时刷新 |

---

### 2.3 AI 模型

| 阶段 | 模型 | 说明 |
|------|------|------|
| **开发阶段** | Ollama + qwen2.5:7b | 本地运行，零成本，响应快 |
| **测试阶段** | OpenAI gpt-4o-mini | 性价比高，效果好 |
| **备选** | 通义千问 / 智谱 GLM | 国产模型，OpenAI 兼容接口 |

**Ollama 配置（application-dev.yml）：**
```yaml
spring:
  ai:
    openai:
      base-url: http://localhost:11434   # Ollama 地址
      api-key: ollama                     # Ollama 不需要真实 key
      chat:
        options:
          model: qwen2.5:7b
      embedding:
        options:
          model: nomic-embed-text         # Ollama 本地 Embedding 模型
```

**OpenAI 配置（application-prod.yml）：**
```yaml
spring:
  ai:
    openai:
      base-url: https://api.openai.com
      api-key: ${OPENAI_API_KEY}
      chat:
        options:
          model: gpt-4o-mini
      embedding:
        options:
          model: text-embedding-3-small
```

### 2.4 文档解析：Apache Tika

| 选项 | 评估 |
|------|------|
| **Apache Tika** ✅ | 支持 1000+ 文件格式，自动检测类型，单依赖搞定 |
| PDFBox | 仅支持 PDF，需额外处理其他格式 |
| POI + PDFBox | 需要多个依赖，配置复杂 |

**决策：** Apache Tika，一个依赖解决所有文档格式。

```xml
<dependency>
    <groupId>org.apache.tika</groupId>
    <artifactId>tika-core</artifactId>
    <version>2.9.2</version>
</dependency>
```

### 3.2 认证：Spring Security + JWT (jjwt)

| 选项 | 评估 |
|------|------|
| **Spring Security + jjwt** ✅ | Spring 官方安全框架 + 最流行的 JWT 库 |
| Sa-Token | 轻量级，API 简洁，但与 Spring Security 集成需要额外工作 |
| 自研 Filter | 简单但不安全，缺少标准安全防护 |

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>
<dependency>
    <groupId>io.jsonwebtoken</groupId>
    <artifactId>jjwt-api</artifactId>
    <version>0.12.6</version>
</dependency>
<dependency>
    <groupId>io.jsonwebtoken</groupId>
    <artifactId>jjwt-impl</artifactId>
    <version>0.12.6</version>
    <scope>runtime</scope>
</dependency>
<dependency>
    <groupId>io.jsonwebtoken</groupId>
    <artifactId>jjwt-jackson</artifactId>
    <version>0.12.6</version>
    <scope>runtime</scope>
</dependency>
```
