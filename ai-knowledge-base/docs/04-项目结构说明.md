# 项目结构说明

> 这份文档告诉你：项目里有哪些东西，每个东西是干什么的，为什么要放在这里

---

## 整体目录结构

```
ai-knowledge-base/
├── pom.xml                                  ← Maven 依赖管理（相当于 package.json）
├── sql/
│   └── schema.sql                           ← 建表 SQL，拿到新数据库直接执行
│
└── src/main/
    ├── java/com/mj/aiknowledgebase/
    │   ├── AiKnowledgeBaseApplication.java  ← 启动类，程序入口
    │   │
    │   ├── common/                          ← 通用组件（所有模块都要用的）
    │   ├── config/                          ← 配置类（Spring 启动时自动加载）
    │   ├── util/                            ← 工具类（纯功能，没有业务逻辑）
    │   ├── controller/                      ← 接口层（接收 HTTP 请求）
    │   ├── service/                         ← 业务层（写业务逻辑）
    │   ├── mapper/                          ← 数据访问层（操作数据库）
    │   └── entity/                          ← 实体类（对应数据库表）
    │
    └── resources/
        └── application.yml                  ← 项目配置文件
```

## 每个包/文件的作用

---

### 1. 启动类

```
AiKnowledgeBaseApplication.java
```

**作用：** 程序从这里开始运行

**包含什么：**
- `@SpringBootApplication` — 告诉 Spring "从这里开始扫描"
- `main()` 方法 — Java 程序的标准入口

**你需要知道：**
- 这个文件一般不改
- 它会自动扫描同包及子包下所有带 `@Component` / `@Service` / `@Controller` / `@Mapper` 的类，把它们注册到 Spring 容器

---

### 2. common — 通用组件

```
common/
├── R.java                       ← 统一响应体
└── GlobalExceptionHandler.java  ← 全局异常处理
```

#### R.java — 统一响应体

**作用：** 让所有接口返回格式统一

**没有它：**
```json
// 接口A返回
{ "user": { "id": 1 } }

// 接口B返回
{ "message": "成功", "result": true }

// 接口C返回
"操作完成"
```

**有了它：**
```json
// 所有接口都返回这个格式
{ "code": 200, "msg": "success", "data": {} }
```

**包含什么：**
- `code` — 状态码（200=成功，400=参数错误，500=服务器错误）
- `msg` — 提示信息
- `data` — 实际数据（泛型，可以是任何类型）
- 静态方法 `R.ok()` 和 `R.error()` — 快速创建响应

**为什么放 common：** 所有 Controller 都要用

---

#### GlobalExceptionHandler.java — 全局异常处理

**作用：** 统一捕获所有异常，返回友好提示

**没有它：**
```
Service 抛异常 → Controller 没捕获 → 用户看到 500 白屏错误页
```

**有了它：**
```
Service 抛异常 → GlobalExceptionHandler 捕获 → 返回 { "code": 400, "msg": "用户名已存在" }
```

**包含什么：**
- `@RestControllerAdvice` — 标记为全局异常处理器
- `@ExceptionHandler(RuntimeException.class)` — 处理 RuntimeException
- `@ExceptionHandler(Exception.class)` — 兜底处理所有异常

**为什么放 common：** 全局只需要一个

---

### 3. config — 配置类

```
config/
├── WebConfig.java           ← 跨域 + 注册 Filter
├── JwtFilter.java           ← JWT 拦截器
├── MyBatisPlusConfig.java   ← MyBatis Plus 配置
└── RedisConfig.java         ← Redis 配置
```

**config 包的特点：**
- 都带 `@Configuration` 注解
- Spring 启动时自动加载，不需要手动调用
- 主要用来"告诉 Spring 一些规则"

---

#### WebConfig.java — Web 配置

**作用：** 两件事

**① 注册 JwtFilter**
```
为什么要"注册"？

Filter 不是 Controller，Spring 不会自动发现它
需要手动告诉 Spring："这个 Filter 要拦截哪些路径"

注册方式：
FilterRegistrationBean<JwtFilter>
  ├── setFilter(jwtFilter)           ← 用哪个 Filter
  ├── addUrlPatterns("/api/*")       ← 拦截哪些路径
  └── setOrder(1)                    ← 执行顺序
```

**② CORS 跨域配置**
```
为什么要配 CORS？

浏览器安全策略：前端 http://localhost:5173 访问后端 http://localhost:8080
会被浏览器拦截，报 CORS 错误

配置后：告诉浏览器"允许跨域访问"
```

---

#### JwtFilter.java — JWT 拦截器

**作用：** 验证每个请求是否登录

**工作流程：**
```
请求进来
    │
    ▼
是白名单路径吗？（/api/register, /api/login）
    │
    ├── 是 → 直接放行
    │
    └── 否 → 从 Header 取 Authorization
              │
              ├── 没有 → 返回 401 未登录
              │
              └── 有 → 解析 JWT
                        │
                        ├── 解析失败/过期 → 返回 401
                        │
                        └── 解析成功 → 取出 userId
                                       存入 request.setAttribute("userId", userId)
                                       放行
```

**关键代码：**
```java
request.setAttribute("userId", userId);   // 存进去
filterChain.doFilter(request, response);  // 放行
```

**后续 Controller 取出来：**
```java
Long userId = (Long) request.getAttribute("userId");
```

**为什么放 config：** 它是请求处理链的一部分，属于"配置"范畴

---

#### MyBatisPlusConfig.java — MyBatis Plus 配置

**作用：** 两件事

**① 分页插件**
```
为什么要配？

MyBatis Plus 的分页功能需要注册插件才能用
不配的话，调用 Page 查询会不生效

配置后就能用：
Page<User> page = new Page<>(1, 10);  // 第1页，每页10条
userMapper.selectPage(page, wrapper);
```

**② 自动填充处理器**
```
为什么要配？

数据库有 create_time 和 update_time 两个字段
不想每次 insert/update 都手动 set

配好后：
Entity 上标 @TableField(fill = FieldFill.INSERT)
→ 插入时自动填入当前时间

Entity 上标 @TableField(fill = FieldFill.INSERT_UPDATE)
→ 插入和更新时都自动填入当前时间
```

---

#### RedisConfig.java — Redis 配置

**作用：** 注册 Redis 操作模板

**包含什么：**
- `StringRedisTemplate` — 操作 Redis 的工具类

**基本用法：**
```java
@Autowired
private StringRedisTemplate redisTemplate;

// 存
redisTemplate.opsForValue().set("token:1", "eyJxxx...", 24, TimeUnit.HOURS);

// 取
String token = redisTemplate.opsForValue().get("token:1");

// 删
redisTemplate.delete("token:1");
```

**为什么要用 Redis 存 Token：**
```
JWT 本身无法主动失效（发出去就管不了了）

解决办法：
登录时 → 生成 Token，同时存 Redis
验证时 → 解析 JWT 成功，还要查 Redis 有没有这个 Token
退出时 → 删 Redis 的 key，Token 立刻失效
```

### 4. util — 工具类

```
util/
└── JwtUtil.java    ← JWT 工具
```

**util 包的特点：**
- 纯功能封装，没有业务逻辑
- 不依赖 Service / Mapper
- 可以在任何地方调用

---

#### JwtUtil.java — JWT 工具类

**作用：** 生成 Token、解析 Token、验证 Token

**提供方法：**
| 方法 | 作用 | 谁调用 |
|------|------|--------|
| `generateToken(userId, username)` | 生成 JWT Token | UserService.login() |
| `parseToken(token)` | 解析 Token，返回 Claims | 内部使用 |
| `getUserId(token)` | 从 Token 取 userId | JwtFilter |
| `getUsername(token)` | 从 Token 取 username | 暂时没用，留着 |
| `isTokenExpired(token)` | 判断是否过期 | 暂时没用，留着 |

**JWT Token 长什么样：**
```
eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOjEsInVzZXJuYW1lIjoiemhhbmdzYW4iLCJleHAiOjE3MjAwMDAwMDB9.xxxxx

由三部分拼接，用 . 分隔：
Header.Payload.Signature
  │       │       │
  │       │       └─ 签名（防止篡改）
  │       └───────── 载荷（存了 userId 和 username）
  └───────────────── 头部（算法信息）
```

---

### 5. controller — 接口层

```
controller/
├── AuthController.java          ← 认证接口（注册、登录）
├── KnowledgeController.java     ← 知识库接口（创建、列表、删除）
└── DocumentController.java      ← 文档接口（上传、列表、详情）
```

**controller 的职责：**
1. 接收 HTTP 请求（参数、文件）
2. 调用 Service 处理业务
3. 返回 R<T> 响应

**controller 的特点：**
- 不写业务逻辑（交给 Service）
- 不操作数据库（交给 Mapper/Service）
- 只做"接参数 → 调 Service → 包返回值"

**请求处理完整链路：**
```
HTTP 请求
    → JwtFilter（验证 Token，取出 userId）
    → Controller（接参数，调 Service）
    → Service（业务逻辑，调 Mapper）
    → Mapper（操作数据库）
    → 返回结果逆向回到 Controller
    → 包装成 R<T> 返回
```

---

### 6. service — 业务层

```
service/
├── UserService.java         ← 用户业务（注册、登录）
├── KnowledgeService.java    ← 知识库业务（CRUD）
└── DocumentService.java     ← 文档业务（上传、查询）
```

**service 的职责：**
1. 写业务逻辑
2. 调 Mapper 操作数据库
3. 抛异常（由 GlobalExceptionHandler 统一处理）

**service 的特点：**
- 可以调其他 Service（如 DocumentService 未来可调 EmbeddingService）
- 可以调 Mapper 做数据库操作
- 不接收 HTTP 请求（那是 Controller 的事）

**示例：UserService.login() 做了什么**
```
1. 用 username 查数据库（调 Mapper）
2. 用 BCrypt 验证密码
3. 生成 JWT Token（调 JwtUtil）
4. 存 Token 到 Redis（调 RedisTemplate）
5. 返回 Token
```

---

### 7. mapper — 数据访问层

```
mapper/
├── UserMapper.java
├── KnowledgeMapper.java
└── DocumentMapper.java
```

**mapper 的职责：**
- 对应一张数据库表
- 提供 CRUD 操作

**为什么只需要一个接口，不需要写实现？**
```
MyBatis Plus 的 BaseMapper<T> 已经内置了：

insert(entity)              → INSERT INTO table ...
deleteById(id)              → DELETE FROM table WHERE id = ?
updateById(entity)          → UPDATE table SET ... WHERE id = ?
selectById(id)              → SELECT * FROM table WHERE id = ?
selectList(wrapper)         → SELECT * FROM table WHERE ...
selectPage(page, wrapper)   → SELECT * FROM table WHERE ... LIMIT ...

都不需要写 SQL，MyBatis Plus 帮你生成
```

**如果需要复杂查询怎么办？**
```
两种方式：

① LambdaQueryWrapper（推荐，类型安全）
   LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
   wrapper.eq(User::getUsername, "zhangsan")
          .gt(User::getId, 10)
          .orderByDesc(User::getCreateTime);

② XML 文件（复杂 SQL 时用）
   在 mapper/ 下新建 UserMapper.xml，写原生 SQL
   当前项目不需要
```

---

### 8. entity — 实体类

```
entity/
├── User.java
├── Knowledge.java
└── Document.java
```

**entity 的职责：**
- 一一对应数据库的表
- 字段名和数据库列名一一对应（驼峰 ↔ 下划线自动转换）

**注解说明：**
```java
@Data                           ← Lombok：自动生成 getter/setter/toString
@TableName("user")             ← 对应数据库的 user 表
@TableId(type = IdType.AUTO)   ← 主键，自增
@TableField(fill = FieldFill.INSERT)        ← insert 时自动填入时间
@TableField(fill = FieldFill.INSERT_UPDATE)  ← insert 和 update 时都自动填入时间
```

**数据库列名 ↔ Java 字段名 映射规则：**
```
数据库          Java             规则
──────────────  ───────────────  ──────────
id              id               相同
user_id         userId           下划线转驼峰（自动）
file_name       fileName         下划线转驼峰（自动）
create_time     createTime       下划线转驼峰（自动）
```

**前提：** application.yml 里配了 `map-underscore-to-camel-case: true`

## 请求从进入到返回的完整流程

```
① 前端/Postman 发请求
   POST http://localhost:8080/api/knowledge
   Header: Authorization: Bearer eyJxxx...
   Body: { "name": "测试" }

② Spring 接收请求，按 URL 找到 Controller

③ JwtFilter 先拦截
   ├── 白名单路径？不是，继续
   ├── 从 Header 取 Authorization
   ├── 调 JwtUtil.parseToken() 解析
   ├── 得到 userId = 1
   └── request.setAttribute("userId", 1)，放行

④ KnowledgeController.create() 执行
   ├── @RequestBody 把 JSON 转成 Map
   ├── request.getAttribute("userId") 取出当前用户
   └── 调 knowledgeService.create(1, "测试", null)

⑤ KnowledgeService.create() 执行
   ├── 创建 Knowledge 对象
   ├── 调 knowledgeMapper.insert(knowledge)
   └── 返回 knowledge

⑥ KnowledgeMapper.insert() 执行
   ├── MyBatis Plus 自动生成 SQL: INSERT INTO knowledge (user_id, name) VALUES (1, '测试')
   ├── 自动填充 create_time 和 updateTime
   └── 执行 SQL，返回影响行数

⑦ 结果逆向回到 Controller
   └── return R.ok("创建成功", knowledge)

⑧ Spring 把 R 对象转成 JSON，返回给前端
   { "code": 200, "msg": "创建成功", "data": { "id": 1, "name": "测试", ... } }
```

## 包之间的依赖关系

```
controller ──调用──▶ service ──调用──▶ mapper
    │                  │                  │
    │                  │                  ▼
    │                  │              entity（数据载体）
    │                  │
    │                  └──调用──▶ util（JwtUtil 等）
    │
    └──使用──▶ common（R, GlobalExceptionHandler）

config ──使用──▶ util（JwtUtil）
所有包 ──扫描──▶ config（Spring 自动加载配置）
```

**规则：**
- Controller 可以调 Service，不能调 Mapper
- Service 可以调 Mapper 和其他 Service
- Mapper 只操作数据库，不调其他层
- Entity 是纯数据对象，不调用任何东西
- Common 和 Util 可以被任何层调用，但不调用业务层
